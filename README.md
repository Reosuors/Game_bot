# ⚡ البوت الخارق - بوت ديسكورد متطور

بوت ديسكورد كامل فيه العاب، ترفيه، ونظام اقتصاد، مبني بـ discord.js v14.

## 🎮 المحتويات

### الالعاب
- **اكس اند او** — ضد لاعب او ضد البوت (بالذكاء الاصطناعي Minimax، شبه مستحيل تفوز عليه 😄)
- **حجر ورقة مقص** — ضد لاعب او البوت
- **مافيا** — لعبة جماعية كاملة (ادوار: مافيا، طبيب، محقق، مواطن) بمراحل ليل/نهار وتصويت وتنسيق عبر الرسائل الخاصة
- **اربعة متصلة (Connect Four)** — ضد لاعب اخر
- **حروف الشنق (Hangman)** — تخمين كلمة حرف بحرف
- **اسئلة ثقافية (Trivia)** — بمكافأة عملات لاول اجابة صحيحة
- **خمن الرقم** — تخمين رقم بين 1-100 بعدد محاولات محدود ومكافأة حسب سرعتك
- **نرد وقلب عملة** — مع امكانية المراهنة

### الترفيه
- الكرة السحرية، نسبة التوافق، نكتة عشوائية، عرض صورة البروفايل

### نظام الاقتصاد والمستويات
- رصيد، مكافأة يومية، عمل، اعطاء عملات، لوحة المتصدرين
- نظام خبرة (XP) تلقائي من الدردشة، يرفع مستواك ويعطيك مكافأة عند كل مستوى جديد

### الادوات
- معلومات العضو، معلومات السيرفر، بينق (سرعة الاستجابة)
- امر `/مساعدة` بقائمة تفاعلية (Select Menu) لعرض كل الاوامر مصنفة

## 🚀 خطوات التشغيل

### 1. انشاء تطبيق البوت
1. روح لـ https://discord.com/developers/applications واضغط **New Application**
2. من تبويب **Bot** اضغط **Add Bot**، وفعّل الخيارات التالية تحت **Privileged Gateway Intents**:
   - `MESSAGE CONTENT INTENT` (ضروري لالعاب حروف الشنق وخمن الرقم ونظام الخبرة)
   - `SERVER MEMBERS INTENT` (لمعلومات العضو)
3. انسخ الـ **Token** من نفس الصفحة (زر Reset Token اذا لزم)
4. من تبويب **OAuth2 > General** انسخ الـ **Client ID**

### 2. تثبيت المتطلبات
```bash
npm install
```

### 3. اعداد بيانات البوت
انسخ ملف `.env.example` وسمّه `.env`، وعبّي البيانات:
```
TOKEN=توكن_البوت
CLIENT_ID=ايدي_التطبيق
GUILD_ID=ايدي_سيرفرك (اختياري، للرفع الفوري اثناء التجربة)
```

### 4. رفع اوامر Slash
```bash
npm run deploy
```
اذا حطيت `GUILD_ID` بيظهر الاوامر فوراً بسيرفرك، واذا تركته فاضي بينرفع عالمياً (يأخذ حتى ساعة).

### 5. تشغيل البوت
```bash
npm start
```

### 6. دعوة البوت لسيرفرك
اذهب لـ **OAuth2 > URL Generator**، اختر Scopes: `bot` و `applications.commands`،
وتحت Bot Permissions فعّل: `Send Messages`, `Embed Links`, `Read Message History`, `Use Slash Commands`.
انسخ الرابط الناتج وافتحه لدعوة البوت.

## ☁️ رفع البوت على استضافة Render

البوت جاهز بالكامل للرفع على Render بدون اي تعديل بالكود. التوكن ومعرفات البوت تدخلها من لوحة تحكم Render نفسها (Environment Variables) وليس داخل اي ملف.

### الخطوات
1. ارفع مجلد المشروع كامل الى مستودع GitHub (Repository) خاص بك.
2. روح لـ https://render.com وسجل دخول، ثم اضغط **New +** واختر **Web Service**.
3. اربط حساب GitHub واختر المستودع اللي رفعت فيه البوت.
4. Render غالباً بيكتشف الاعدادات تلقائياً من ملف `render.yaml` الموجود بالمشروع، وإلا عبّي يدوياً:
   - **Environment:** `Node`
   - **Build Command:** `npm install`
   - **Start Command:** `npm start`
   - **Plan:** Free (كافي للبوت)
5. تحت قسم **Environment Variables** ضيف المتغيرات التالية (بدون ما تلمس اي ملف بالكود):
   | المتغير | القيمة |
   |---|---|
   | `TOKEN` | توكن البوت من Discord Developer Portal |
   | `CLIENT_ID` | معرف التطبيق (Application ID) |
   | `GUILD_ID` | (اختياري) معرف سيرفرك اذا تبي الاوامر تظهر فوراً بدل ما تنتظر ساعة |
6. اضغط **Create Web Service**. Render بيبني ويشغل البوت تلقائياً.
7. البوت يرفع اوامر Slash تلقائياً بنفسه اول ما يشتغل (مافيه خطوة يدوية اضافية).

### ملاحظات خاصة باستضافة Render
- خدمات الويب على Render (حتى المجانية) لازم "تستمع" على منفذ HTTP، لهذا مضاف بالمشروع سيرفر بسيط (`utils/keepAlive.js`) يخلي Render يعتبر البوت شغال — هذا لا يؤثر على شغل البوت الاساسي إطلاقاً.
- الخطة المجانية على Render توقف الخدمة اذا ما وصلها طلب HTTP لمدة، وتعيد تشغيلها عند اول طلب — ممكن يسبب تأخر بسيط برد البوت بعد فترة خمول. اذا تبي البوت يفضل شغال 24/7 بدون توقف، تحتاج خطة مدفوعة او استخدام خدمة "Background Worker" بدل "Web Service".
- بيانات الاقتصاد والمستويات (`data/*.json`) تُخزن على القرص المحلي للخدمة، وفي الخطة المجانية القرص غير دائم فينمسح عند كل رفعة جديدة (Deploy). اذا تبي البيانات تضل محفوظة بشكل دائم، اخبرني واقدر اربطها بقاعدة بيانات خارجية مجانية (مثل MongoDB Atlas).

## 📁 هيكلية المشروع
```
discord-bot/
├── index.js                 # نقطة تشغيل البوت
├── deploy-commands.js        # رفع اوامر Slash
├── commands/
│   ├── games/                # اكس-او، حجر-ورقة-مقص، مافيا، اربعة-متصلة، الخ
│   ├── fun/                   # الكرة السحرية، توافق، نكتة، صورة البروفايل
│   ├── economy/                # رصيد، يومي، اعمل، اعطاء، المتصدرين
│   └── utility/                 # مساعدة، معلومات العضو/السيرفر، بينق
├── events/
│   ├── ready.js                # حدث استعداد البوت
│   ├── interactionCreate.js     # معالج الاوامر والازرار والقوائم
│   └── messageCreate.js          # حروف الشنق، خمن الرقم، نظام الخبرة
├── utils/
│   ├── database.js               # تخزين JSON بسيط (اقتصاد + مستويات)
│   ├── embeds.js                  # قوالب Embed موحدة
│   ├── mafiaEngine.js              # محرك مراحل لعبة المافيا
│   ├── deployCommands.js            # رفع اوامر Slash تلقائياً عند التشغيل
│   └── keepAlive.js                  # سيرفر HTTP بسيط لاستضافات مثل Render
├── render.yaml                          # اعدادات نشر جاهزة لـ Render
└── data/                                  # ملفات JSON تتولد تلقائياً (اقتصاد/مستويات)
```

## 💡 ملاحظات مهمة
- التخزين حالياً عبارة عن ملفات JSON محلية (بسيطة وسهلة، مناسبة للسيرفرات المتوسطة). اذا احتجت قاعدة بيانات حقيقية (MongoDB/PostgreSQL) قدر اضيفها لاحقاً.
- لعبة المافيا تحتاج من كل لاعب انه يكون خاصه (DM) مفتوح للبوت عشان تجيه رسائل الدور والافعال الليلية.
- كل الالعاب تُدار بالذاكرة (Map) — اذا اعدت تشغيل البوت، الالعاب النشطة تنمسح.
- تقدر تضيف اي امر جديد بسهولة: سوي ملف جديد داخل المجلد المناسب في `commands/` بنفس النمط (data + execute) ثم شغّل `npm run deploy`.
